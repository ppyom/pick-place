# Design System

Pick!Place는 장소를 발견하고 기록하며 공유하는 서비스입니다.

브랜드 컬러는 하나의 Primary를 중심으로 사용하며, Secondary는 보조(Accent) 컬러로 활용합니다.

- 🌿 자연스러운 분위기
- ☕ 따뜻하고 편안한 감성
- 📍 장소와 사진이 돋보이는 UI
- ✨ 미니멀한 디자인

**Font**: SUITE

## Colors — Semantic

실제 컴포넌트에서 사용하는 의미 기반 토큰입니다.

### Surface

<ColorPalette>
  <ColorItem
    title="Surface"
    colors={{
      Background: 'var(--color-surface-background)',
      Brand: 'var(--color-surface-brand)',
      'Brand-Subtle': 'var(--color-surface-brand-subtle)',
      Secondary: 'var(--color-surface-secondary)',
      Card: 'var(--color-surface-card)',
      Elevated: 'var(--color-surface-elevated)',
      Disabled: 'var(--color-surface-disabled)',
      Inverse: 'var(--color-surface-inverse)',
    }}
  />
</ColorPalette>

> `Brand`는 꽉 찬 브랜드색(버튼 등 강조용), `Brand-Subtle`은 옅은 브랜드색(카드/배경 등 은은한 용도)입니다. 둘 다 라이트/다크 모드와 무관하게 고정값입니다.

### Text

<ColorPalette>
  <ColorItem
    title="Text"
    colors={{

> `Inverse`는 테마(Light/Dark)에 따라 반전되는 배경 위에서 쓰는 텍스트색입니다. `On-Brand`는 고정된 브랜드 배경(`Surface/Brand`) 위에서만 쓰는 전용 텍스트색으로, 테마와 무관하게 항상 White입니다.

## Naming Convention

- **Primitive**: `Primary/500`, `Neutral/200`, `Success/500`
- **Semantic**: `Surface/Brand`, `Text/On-Brand`, `Status/Success/Text`
- **Component**: `Button/Primary/Background`, `Input/Border`, `Chip/Selected`

Component 토큰은 원칙적으로 Semantic만 참조합니다. 재사용성이 낮아 Semantic으로 승격하기 어려운 값만 예외적으로 Primitive를 직접 참조하며, 이 경우 사유를 문서에 명시합니다.
